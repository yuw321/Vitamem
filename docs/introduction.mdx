# What is Vitamem?

Vitamem is a TypeScript library that gives AI health companions **long-term memory that understands the rhythm of care**.

Most memory libraries treat every conversation message the same -- embedding everything, storing everything equally. Vitamem is different. It models the natural lifecycle of a care relationship:

- An **active** session where the user is checking in
- A **cooling** period between sessions (the relationship is still live)
- A **dormant** break where facts are compressed and embedded once
- A **closed** state when the relationship ends

This lifecycle approach makes Vitamem ideal for health companions, wellness chatbots, and counseling assistants -- AI that remembers users the way a caring friend would.

## 3 Lines to Long-Term Memory

```typescript
import { createVitamem } from "vitamem";

const mem = await createVitamem({
  provider: "openai",
  apiKey: process.env.OPENAI_API_KEY!,
  storage: "ephemeral",
});
```

No manual adapter wiring. No boilerplate. Pass a provider name, an API key, and a storage backend -- Vitamem handles the rest. Built-in adapters are available for OpenAI, Anthropic, and Ollama.

## Why Vitamem

Most AI memory systems embed every message as it arrives. This creates three problems:

1. **Cost.** Embedding calls on every message add up fast, especially in long health consultations.
2. **Noise.** Raw messages contain greetings, follow-up questions, and filler that dilute the memory store. Searching through noisy embeddings returns poor results.
3. **Duplication.** A user who mentions "I take metformin" in five sessions gets five near-identical memories, wasting storage and confusing retrieval.

Vitamem takes a different approach: **lifecycle-aware memory extraction.**

Instead of embedding messages in real time, Vitamem waits until a conversation session goes dormant. At that point, it sends the full conversation to the LLM with an extraction prompt, gets back structured facts (like "Has Type 2 diabetes" and "Takes metformin 500mg twice daily"), embeds those facts, and deduplicates them against existing memories using cosine similarity.

The result:

- **5-20x fewer embedding calls** -- computed once per session on extracted facts, not on every message
- **Cleaner memories** -- structured facts instead of raw conversational noise
- **Automatic deduplication** -- genuinely new facts are preserved, redundant ones are discarded
- **Better retrieval** -- searching over concise facts produces more relevant results than searching over chat messages

This is how a caring professional remembers a patient: not by replaying every word of every appointment, but by maintaining a clean, updated record of the things that matter.

## The Core Insight

Embeddings are expensive. Most systems embed every message as it arrives. Vitamem embeds **once per thread**, at the moment the conversation goes dormant -- on a compressed, deduplicated summary of extracted facts rather than raw messages.

This results in **5-20x fewer embedding calls** than naive per-message approaches while producing better, more meaningful memories.

## What Vitamem Stores

When a thread goes dormant, Vitamem extracts and stores **facts about the user**:

```
"Has Type 2 diabetes"           (source: confirmed)
"Takes metformin daily"         (source: confirmed)
"Exercises 3 times per week"    (source: inferred)
"Prefers morning check-ins"     (source: inferred)
```

These facts are embedded as vectors and deduplicated against existing memories using cosine similarity. Later, when the user returns, the AI can retrieve relevant facts and respond like it actually remembers them.

## Use Cases

- **AI health companions** -- remembers symptoms, medications, conditions, and health goals
- **Counseling assistants** -- maintains emotional context and tracks therapeutic progress
- **Wellness apps** -- builds a richer user model over time without expensive per-message processing
- **Chronic condition trackers** -- persists health history across long gaps between check-ins
